# Security Policy

## Supported Versions

We actively maintain and support the latest release versions of the platform. Please verify you are running the latest stable build before reporting any security concerns.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability or security flaw in the platform, please report it to us responsibly:

1. **Do not** open a public GitHub issue.
2. Report your findings via email directly to the repository maintainer (details can be found in `pyproject.toml` or author info).
3. Include a detailed description of the vulnerability, a proof of concept (PoC), and steps to reproduce.

We will review your submission and coordinate a patch/fix as quickly as possible.
